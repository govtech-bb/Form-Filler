# Plan: Remove unused `tabs` permission (CWS rejection — Purple Potassium)

## Goal

Get Form Filler accepted on the Chrome Web Store by removing the `tabs`
permission that triggered the "Use of Permissions" rejection (violation ref
*Purple Potassium*, Jun 5 2026), then ship a resubmittable build.

## Background

The Web Store rejected v1.0.0 because the manifest declares `tabs`, which the
extension does not use. The `tabs` permission *only* gates the `url`,
`pendingUrl`, `title`, and `favIconUrl` properties of `Tab` objects. Tracing
every `chrome.tabs` call in `src/background/index.ts`:

- `chrome.tabs.sendMessage(...)` — never requires `tabs`.
- `chrome.tabs.query({ active: true, currentWindow: true })` (lines 183, 202,
  217) — only `tab.id` is read; reading `id` does not require `tabs`.

No code touches `url`/`pendingUrl`/`title`/`favIconUrl`, so removing `tabs`
changes no behaviour. The reviewer is correct.

The other three permissions are all justified and stay:

- `activeTab` — grants `chrome.scripting.executeScript` temporary host access to
  the current tab when the user invokes the extension (popup click or keyboard
  command). Required by `ensureContentScript`.
- `scripting` — required for `executeScript` itself.
- `storage` — required for `chrome.storage.local`.

## Approach

One-line manifest fix plus a version bump (the Web Store will not accept a
re-upload at the same version `1.0.0`), then a clean rebuild so the uploaded
package reflects the change.

Alternative considered — *appeal instead of fix*: rejected. The permission is
genuinely unused, so removing it is faster and unambiguous than arguing it.

## Scope

- Remove `"tabs"` from the `permissions` array in `manifest.json`.
- Bump version `1.0.0` → `1.0.1` in `manifest.json` (source of truth — the build
  reads `manifest.json` via `crx({ manifest })`).
- Bump version `1.0.0` → `1.0.1` in `package.json` for consistency.
- Rebuild `dist/` with `pnpm build` so the upload package is current.

## Files

- `manifest.json` — drop `"tabs"`; `version` → `1.0.1`.
- `package.json` — `version` → `1.0.1`.
- `dist/` — regenerated by the build; **not** tracked in git (it is
  `.gitignore`d), so it produces no diff to commit.

## Verify

- `pnpm test` — stays green (no logic changed).
- `pnpm build` — completes cleanly.
- Inspect built `dist/manifest.json`: `permissions` is exactly
  `["activeTab", "scripting", "storage"]` and `version` is `1.0.1`.
- Load the unpacked `dist/` in Chrome and confirm a normal fill still works
  (popup fill + `Ctrl+Shift+F`), proving the removed permission was never needed.
- Package `dist/` and resubmit to the Web Store.
